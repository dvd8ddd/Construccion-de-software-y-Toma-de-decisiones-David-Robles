Preguntas

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Intertnet es la Red global de computadoras interconectadas que permite la comunicación y el intercambio de información.

La World Wide Web es un sistema de documentos e hipervínculos que funciona sobre Internet y permite acceder a contenido mediante navegadores.

2. ¿Cuáles son las partes de un URL?

El Protocolo (https)

El Dominio (www.callofduty.com)

El Puerto (443)

La Ruta (/restaurantes/buscar)

Quary string (?fecha=2025-01-15)

El Fragmento (el navegador lo usa para hacer scroll)

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Get: Obtener datos

Head: Solo las cabeceras pero sin cuerpo

Post: crear un recurso nuevo

Put: Reemplazar un recurso completo

Patch: Actualizar parcialmente

Delete: eliminar

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Post, porque los datos no van en la URL y son más seguros

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que todo salió bien

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Un error 404 significa que la página no fue encontrada, puede pasar por una URL mal escrita o porque la página fue eliminada. El desarrollador tiene que manejar este error correctamente

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es responsabiliad y que un error 500 es un error interno asi que es responsabiliad del desarrollador

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

HTML deprecated significa etiquetas obsoletas, algunos elementos son font, center, b, i

12. Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de un sistema son las etapas por las que pasa un sistema: planeación, análisis, diseño, implementación y mantenimiento

El ciclo de desarrollo es la metodologia que utiliza para construir el sistema como modelo cascada.
